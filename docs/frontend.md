# Frontend Architecture

The Semantica frontend is a modern single-page application built using Vite and React, heavily leveraging modern React Hooks for state management to handle complex Server-Sent Event (SSE) streams robustly.

## Tech Stack
- **Framework**: React 19 + Vite
- **Styling**: Tailwind CSS + raw Vanilla PostCSS setup
- **Icons**: `lucide-react`
- **Toasts**: `react-hot-toast`

## Core Components
- `<App />` (`src/App.jsx`): Mounts the `HomePage` view along with the global `ToastContainer`.
- `<HistoryPanel />`: Used to view the past runs generated by agents.

## State Management
The most complex part of the frontend is maintaining synchronous visuals for SSE event-streams that dictate agent status, such as:
1. Emitting `analyzing`, `planning`, `coding`, `executing`, and `verifying` states contextually.
2. Building an asynchronous UI state via hooks such as `useAgentRun.js`. The hook processes the event loop and yields react variables.

## Build Scripts
The package definitions (`package.json`) run the following:
- `npm run dev`: Boots the local Vite DEV server.
- `npm run build`: Bundles for production.
- `npm run lint`: Validates files against ES lint guidelines.

### API Proxy Configuration
In `vite.config.js`, Vite proxies any requests directed to `/api/*` across to the local FastAPI port (`http://localhost:8000/api`). This ensures clean CORS logic during local development.
